Web Development Concepts

Web Servers

When a client goes to a webpage, index.html is the page the server will send back to show the client. The index.html file is the name of the resource on the server. As a convention, most web servers are configured so that the request for / is mapped to the file at the path /index.html. This file is called the designated homepage on the web server. In almost all real-world web applications, the file index.html in the root directory is called the home page of the web application, and the web application is configured to return this file whenever a GET request receives the URL. Index.html is the default name for a home page of a website hosted on an Apache web server.

Frontend Design

Good frontend design is making the web page suitable to the users' experience and purpose of the web page. It includes adapting the viewing environment to the different formats users may use like a mobile phone to big screen display. The graphical user interface needs to be simple to use and intuitive. The organization of the web page should be similar to the front page of a newspaper. The most relevant topics should be positioned to the top right, in order to catch the users' eye first. Usability is about how easy it is for a customer to do one thing such as making a purchase or searching for a topic of interest.

The Five "E"s of Usability

Effective
It is easy for users to meet their goal(s). The main messages are clear.
Efficient
The user can get through the website with the fewest clicks as possible.
Easy to navigate
For first time users, the website should be intuitive to get around. The main topics should be main headers.
Error-free
The website needs to be tested for all the possible results that a customer will want to achieve.
Enjoyable to use, or engaging
The overall experience needs to be hassle-free and give what the customer wants.

Page layout tags are block-level elements such as paragraph, headers, lists, navigation, tables and many more. There are two categories called block level elements or inline level elements that contribute to the flow control in HTML. A Block-level element occupies the entire horizontal space, and vertical space equal to the height of its contents, thereby creating a “block”. On the other hand, inline elements can start anywhere in a line. Browsers typically display the block-level element with a newline both before and after the element. A block-level element always starts on a new line and takes up the full width available. Block-level elements may appear only within a body element.

We create links from one section or page to another using the anchor element. The content between the opening and closing anchor element tags should describe the link when someone clicks to navigate to it. The anchor element's HREF attribute, creates a hyperlink to web pages, files, email addresses, locations on the same page, or anything else a URL can address. Content within each anchor element should indicate the link's destination. If the HREF attribute is present, pressing the enter key while focused on the anchor element will activate it. It is possible to use the anchor element's HREF attribute to link to a specific location within a document, rather than just the top of the document. Text anchor is an attribute to elements such as TEXT, TREF, TSPAN and more. It is used to align (start-, middle- or end-alignment) a string of pre-formatted text or auto-wrapped text and is applied to each individual text chunk within a given element. The navigation HTML element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes. The navigation element is intended only for a major block of navigation links. A document may have several navigation elements, for example, one for site navigation and one for inside the same page navigation.
